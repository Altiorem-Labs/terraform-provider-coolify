# coolify_project

Manages a Coolify project resource. Projects in Coolify are top-level containers that organize your applications, environments, and resources.

## Example Usage

### Basic Project

```hcl
resource "coolify_project" "example" {
  name = "my-awesome-project"
}
```

### Project with Description

```hcl
resource "coolify_project" "webapp" {
  name        = "webapp-project"
  description = "Main web application project for production workloads"
}
```

### Multiple Projects

```hcl
resource "coolify_project" "frontend" {
  name        = "frontend-services"
  description = "Frontend applications and static sites"
}

resource "coolify_project" "backend" {
  name        = "backend-services"
  description = "API services and microservices"
}

resource "coolify_project" "data" {
  name        = "data-platform"
  description = "Data processing and analytics services"
}
```

## Argument Reference

The following arguments are supported:

* `name` - (Required) The name of the project. Must be unique within your Coolify instance.
* `description` - (Optional) A description for the project to help identify its purpose.

## Attribute Reference

In addition to all arguments above, the following attributes are exported:

* `id` - The unique identifier (UUID) for the project. This is automatically generated by Coolify.

## Import

Projects can be imported using their UUID:

```bash
terraform import coolify_project.example 12345678-1234-1234-1234-123456789012
```

## Notes

- Project names must be unique within your Coolify instance
- Once created, projects serve as containers for environments and applications
- Deleting a project will also delete all associated environments and applications
- The project ID is required when creating environments using `coolify_environment` resource
